# TreasuryYield Frontend

A beautiful Next.js 14 Web3 application powering the TreasuryYield platform.

## Architecture

Built utilizing:
- **Next.js 14 App Router**: Server-side rendering (SSR) and seamless layouts.
- **Wagmi / viem**: React Hooks for reading from `YieldVault` and `TBillToken` smart contracts.
- **RainbowKit**: High-end Web3 wallet connection modal.
- **TailwindCSS + Magic UI + framer-motion**: Deep emerald/green translucent glass-morphism aesthetic perfectly aligned with a "Treasury/Yield" narrative.
- **Recharts**: For dynamic rendering of 6-month historical T-Bill trailing APYs.

## Development

Ensure you are using Node `v18.x` or later.

1. **Install Dependencies**
   ```bash
   cd frontend
   npm install
   ```

2. **Configure Web3 Contracts**
   - The contract ABIs and Addresses are hardcoded into `src/config/contracts.ts`.
   - Update these immediately after deploying to a new environment (Testnet/Mainnet/Anvil).

3. **Start the Development Server**
   ```bash
   npm run dev
   ```
   Open `http://localhost:3000` to view the application.

4. **Production Build**
   ```bash
   npm run build
   ```

## Integrations

The primary state integration is inside `src/app/page.tsx`, directly calling the standard ERC4626-like methods generated by our contracts:
- `YieldVault.previewDeposit`
- `YieldVault.previewWithdraw`
- `TBillToken.getAssetPrice`

This architecture allows the UI to predict the precise mathematical return ratios to the user entirely on-chain without client-side floating point calculation errors.
